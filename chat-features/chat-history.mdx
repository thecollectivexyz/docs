## Overview

As an administrator, you can view chat history to:
- Monitor AI usage across the organization
- Help troubleshoot issues users report
- Review conversation quality
- Understand how teams are using NTS Vortex

## Common Admin Tasks

### Investigating Issues

When a user reports a problem:

<Steps>
  <Step title="Get details">
    Ask the user for:
    - Approximate time of the issue
    - What they were asking about
    - The model they were using
  </Step>

  <Step title="Find the conversation">
    Use search and filters to locate the specific conversation.
  </Step>

  <Step title="Review the exchange">
    Look at:
    - What the user asked
    - How the AI responded
    - Any errors or unexpected behavior

    <Frame caption="Investigating an issue">
      <img src="/images/placeholders/chat-features/investigate-issue.png" alt="Reviewing a problematic conversation" />
    </Frame>
  </Step>

  <Step title="Take action">
    Based on findings:
    - Adjust AI settings if needed
    - Update knowledge base if information was wrong
    - Report bugs if there's a system issue
  </Step>
</Steps>

### Monitoring Usage Patterns

Review history regularly to understand:

- Which models are most used
- Common types of questions
- Areas where AI struggles
- Power users and their use cases

<Frame caption="Usage patterns">
  <img src="/images/placeholders/chat-features/usage-patterns.png" alt="Dashboard showing usage statistics" />

### Quality Assurance

Periodically review conversations to:

- Ensure AI responses are accurate
- Check that agents are performing as expected
- Identify training opportunities
- Find knowledge base gaps

## Privacy Considerations

<AccordionGroup>
  <Accordion title="When to access chat history">
    Access chat history only when:
    - Troubleshooting reported issues
    - Performing regular quality reviews
    - Investigating security concerns
    - Required for business purposes
  </Accordion>

  <Accordion title="What to avoid">
    Don't:
    - Browse chat history out of curiosity
    - Share conversation contents unnecessarily
    - Use information for unrelated purposes
    - Access history without legitimate reason
  </Accordion>

  <Accordion title="Handling sensitive information">
    If you find sensitive information:
    - Follow your organization's data handling policies
    - Report security concerns to appropriate parties
    - Consider if retention settings need adjustment
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Can't find a conversation">
    - Expand the date range in filters
    - Try different search terms
    - Check if it was in a different model/agent
    - The conversation may have been deleted
  </Accordion>

  <Accordion title="History not loading">
    - Refresh the page
    - Check your internet connection
    - There may be a lot of data - wait for it to load
    - Try narrowing filters to reduce data
  </Accordion>

  <Accordion title="Missing conversations">
    - Check retention settings - old conversations may be auto-deleted
    - Verify you have admin permissions
    - The user may have deleted the conversation
  </Accordion>
</AccordionGroup>
