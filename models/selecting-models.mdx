## Overview

NTS Vortex can connect to many AI models, but not all need to be available to every user. This guide explains how to manage which models are visible and set defaults.

<Frame caption="Model management interface">
  <img src="/images/placeholders/models/model-management.png" alt="Admin interface for managing models" />
</Frame>

## Understanding Model Options

Different AI models have different strengths:

| Model Type | Strengths | Best For | Cost |
|------------|-----------|----------|------|
| **GPT-4o** | Reasoning, general knowledge | Complex questions, analysis | Higher |
| **GPT-4o-mini** | Fast, capable | Most daily tasks | Lower |
| **Claude 3.5 Sonnet** | Documents, nuance | Long documents, careful work | Medium |
| **Claude 3 Haiku** | Very fast | Quick responses | Lowest |
| **Gemini Pro** | Long context, multimodal | Large documents, images | Medium |
| **Mistral** | Speed, efficiency | Quick tasks | Lower |
| **Llama 3** | Open source, capable | General tasks | Medium |

## Enabling and Disabling Models

Control which models users can see:

<Steps>
  <Step title="Find the model">
    Locate the model you want to enable or disable in the models list.
  </Step>

  <Step title="Toggle visibility">
    Click the toggle or checkbox to enable/disable the model for users.

    <Frame caption="Toggling model visibility">
      <img src="/images/placeholders/models/toggle-model.png" alt="Toggle switch for enabling/disabling a model" />
    </Frame>
  </Step>

  <Step title="Save changes">
    Click **Save** to apply your changes.
  </Step>
</Steps>

<Tip>
Don't overwhelm users with too many options. Enable 3-5 well-chosen models that cover different use cases.
</Tip>

## Setting a Default Model

The default model is what users see when they start a new chat:

<Steps>
  <Step title="Find default model settings">
    Look for **Default Model** or **Default Settings** in the models configuration.

    <Frame caption="Default model setting">
      <img src="/images/placeholders/models/default-model-setting.png" alt="Default model configuration option" />
    </Frame>
  </Step>

  <Step title="Select the default">
    Choose which model should be the default. Consider:
    - A good balance of capability and cost
    - Appropriate for most common use cases
    - Fast enough for a good user experience

    <Frame caption="Selecting default model">
      <img src="/images/placeholders/models/select-default.png" alt="Dropdown to select default model" />
    </Frame>
  </Step>

  <Step title="Save">
    Click **Save** to set the default.
  </Step>
</Steps>

<Info>
Users can still switch to other enabled models - the default just sets the starting point.
</Info>

## Model Parameters

Adjust how models behave:

<Frame caption="Model parameters">
  <img src="/images/placeholders/models/model-parameters.png" alt="Model parameter settings" />
</Frame>

### Temperature

Controls creativity vs. consistency:
- **0.0 - 0.3**: Very consistent, factual
- **0.4 - 0.6**: Balanced
- **0.7 - 1.0**: More creative, varied

### Max Tokens

Maximum response length:
- Lower values = shorter, faster responses
- Higher values = longer, more detailed responses

### Context Length

How much conversation history the model remembers:
- Important for long conversations
- Higher values use more resources

## Testing Models

Before enabling models for all users, test them:

<Steps>
  <Step title="Enable the model for yourself">
    Make sure you can access the model.
  </Step>

  <Step title="Test various scenarios">
    Try:
    - Simple questions
    - Complex requests
    - NTS-specific questions
    - Tasks with knowledge base
  </Step>

  <Step title="Evaluate">
    Consider:
    - Response quality
    - Speed
    - Appropriateness for your users
  </Step>

  <Step title="Enable for others">
    Once satisfied, enable for all users.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Model doesn't appear in dropdown">
    - Verify the model is enabled in admin settings
    - Check that the API connection is working
    - Refresh the page
    - Log out and back in
  </Accordion>

  <Accordion title="Model errors or won't respond">
    - Check API key validity
    - Verify you have credits/quota
    - Try a different model
    - Check provider status pages
  </Accordion>

  <Accordion title="Model is slow">
    - Some models are inherently slower
    - Check your internet connection
    - Consider using a faster model for time-sensitive tasks
  </Accordion>

  <Accordion title="Responses are cut off">
    - Increase max tokens setting
    - The model may have hit its context limit
    - Start a new chat to clear context
  </Accordion>
</AccordionGroup>
